Vegetarian Pizza -- A Typology¶

Pizza¶

I love me a good pizza slice. Cheesy. Crunchy. Satisfying.

So it was a big surprise when I moved to California's Bay Area and had my first encounter with the only-vegetarian pizza place that would change my life: The Cheeseboard. I took one bite of their pizza and have hated every other pizza place since.

But I'm under quarantine right now, which means I can't visit The Cheeseboard, which means I must make my own.

My background is one of content product analysis, so I'm familiar with using multidimensional scaling (MDS) to compare similarities or differences between two sets.

So I wondered could use this tool to help me define "vegetarian pizza master recipes" based on pizza features?

As it turns out: yes. Yes, I could.

CRISP-DM Process¶

Business Understanding: From the menus, we can learn how these pizzas are but together: how are ingredients combined, when are they used, when are they not used, etcetera. Our explicit goal is to understand:

General guidelines for mixing-and-matching ingredients Any vegetarian pizza no-no Given these pizzas are vegetarian, do they cluster around some similarities given the season?

Data Understanding：The scrapped dataset consists of 490 lines of code that fit the description passed on to BeautifulSoup for a menu. That being said, the data also has non-menu lines, blank lines, and has extra information such as the schedule before the pizza.

Prepare Data: Data needs cleaning, dropping NAN values, feature engineering, and tidying up. See below for details.

Data Modeling: Used correlations to find different connections between different pizza features. Created distance matrix to compare similarities and differences of pizzas. Explored different SciKit ML Models to see how it might go about selecting a season for a pizza given its ingredients.

Evaluate the Results: Result and discussion are published here: https://medium.com/@adenipx/the-secrets-of-vegetarian-pizza-decdb913a90?sk=f7b5c1bcf19c3243cb1783a10e01a5fa

# shoutout: https://stackoverflow.com/questions/16139306/determine-season-given-timestamp-in-python-using-datetime

Y = 2000 # dummy leap year to allow input X-02-29 (leap day)
seasons = [('winter', (date(Y,  1,  1),  date(Y,  3, 20))),
           ('spring', (date(Y,  3, 21),  date(Y,  6, 20))),
           ('summer', (date(Y,  6, 21),  date(Y,  9, 22))),
           ('autumn', (date(Y,  9, 23),  date(Y, 12, 20))),
           ('winter', (date(Y, 12, 21),  date(Y, 12, 31)))]

def get_season(now):
    if isinstance(now, datetime):
        now = now.date()
    now = now.replace(year=Y)
    return next(season for season, (start, end) in seasons
                if start <= now <= end)
